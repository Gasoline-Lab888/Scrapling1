# 外贸品牌资料抓取方案（ESBE 示例）

这个目录提供一个“可参考、可扩展”的抓取流程：

1. 抓取 ESBE 产品页。
2. 下载产品说明书 PDF。
3. 提取产品标题 + 图片，生成产品树（JSON + Markdown）。
4. 每 2 周通过 GitHub Actions 自动运行一次。
5. 可选：自动上传到 Google Drive。

## 文件说明

- `esbe_pipeline.py`：主脚本。
- `.github/workflows/esbe_research.yml`：定时任务（每 2 周）。

## 本地运行

```bash
pip install -e .
pip install pydrive2
python examples/brand_research/esbe_pipeline.py
```

输出目录：`data/esbe/`

- `products_tree.json`
- `products_tree.md`
- `images/`
- `docs/`

## GitHub Actions 自动运行

工作流默认：
- 定时：每 2 周（周一 UTC 03:00）
- 手动：支持 `Run workflow`

> 说明：GitHub 免费版总时长约 2000 分钟/月。当前任务每 2 周跑 1 次，通常占用很低。后续品牌变多时，建议：
> 1) 把每个品牌拆成独立 job；
> 2) 控制单次最大页面数；
> 3) 使用缓存减少重复下载；
> 4) 只抓“增量更新”链接。

## 为什么你在桌面端“无法应用/无法继续”

你截图里底部提示：`此任务创建于 Scrapling1，可能无法直接应用。`

这通常不是代码问题，而是**会话绑定的工作目录不一致**：

- 这条任务是在某个仓库/工作区上下文里创建的（例如云端 `Scrapling1`）。
- 你现在在桌面端打开的是另一个本地路径或未绑定同一个 Git 仓库。
- 所以客户端无法把“云端任务的补丁”直接应用到当前本地文件。

### 桌面端可执行的修复方案（推荐）

1. 在桌面端重新打开**同一个仓库路径**（确保有 `.git`，且远程是同一个项目）。
2. 在该本地仓库里新建任务（不要复用云端旧任务）。
3. 在新任务里粘贴同样需求，并明确写：
   - `请直接修改当前本地仓库文件并提交`。
4. 若仍提示不能应用，先 `git pull` 同步，再让客户端重新生成补丁。

### 桌面端要调用浏览器/插件的配置建议

1. 使用桌面端的“插件/MCP”面板，启用浏览器自动化相关插件（Playwright/Browser 类）。
2. 在任务里明确写：
   - `请使用浏览器插件访问页面并抓取动态内容`。
3. 若企业网络有限制，给桌面端配置代理（HTTP_PROXY/HTTPS_PROXY）。
4. 在本地先验证浏览器依赖（例如 Playwright 浏览器内核）是否安装完整。

### 兼容你当前场景的工作方式

- **开发与调试**：在桌面端跑（方便浏览器插件、可视化调试）。
- **定时与归档**：交给 GitHub Actions 跑（两周一次，自动上传 artifact/Drive）。

这样你既能在桌面端灵活调试，也能保持云端稳定自动化。

## 你说“桌面写的是云端运行”——这其实是正常的

桌面客户端可以有两种执行模式：

1. **云端运行模式**：代码实际在远程容器里执行。
2. **本地运行模式**：代码在你电脑本地仓库执行。

你现在看到“任务创建于 Scrapling1，可能无法直接应用”，说明当前会话更偏向**云端工作区上下文**，而不是你当前本地目录。

### 如果你想继续用“桌面 + 云端运行”

- 保持同一个云端仓库会话，不要切换到另一个本地目录再直接点“应用”。
- 需要落地到本地时，用 `git pull` 从远程分支同步云端已提交内容。

### 如果你想改成“桌面 + 本地运行（可直接调浏览器插件）”

1. 在桌面端切到本地运行模式（选择本地仓库目录）。
2. 在本地仓库重新创建任务，让补丁直接作用于本地文件。
3. 再启用浏览器/MCP插件进行动态抓取调试。

## 配置 Google Drive 上传（可选）

在 GitHub 仓库 `Settings -> Secrets and variables -> Actions` 中添加：

1. `GDRIVE_SERVICE_ACCOUNT_JSON`
   - 内容为 Google Service Account 的 JSON 全文。
2. `GDRIVE_FOLDER_ID`
   - 目标 Google Drive 文件夹 ID。

并且把该 Service Account 邮箱加入该文件夹协作者（编辑权限）。
